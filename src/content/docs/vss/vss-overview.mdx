Virtual Script Storage (VSS) is Panda's platform for securely storing and distributing Roblox Lua scripts.

## What Is VSS?

<SectionMarker icon={Target} label="Core Concept" />

VSS is designed to handle the end-to-end script pipeline for you:

- Script obfuscation
- Library injection
- Version control and rollback
- Script distribution via loadstring URLs

## Free Features

<SectionMarker icon={Zap} label="Included At No Cost" />

Available without paid upgrades:

- `IB1` obfuscation (free)
- `wYnfuscator` (beta, free)
- `None` (plain script distribution)
- Version history tracking and rollback
- Library injection: `V4-HyperGuard`, `V3`, `V2.5`, or `None`

## Unlocking Luraph

<SectionMarker icon={Unlock} label="Verification Path" />

Luraph is a premium obfuscator and requires verification.

Verification requirements:

- 1,000+ Discord server members
- Active Panda Key System usage

Verification methods:

1. Invite the Panda bot to your Discord server (automatic verification)
2. Manual verification by a Panda moderator

After verification, unlocks include:

- Unlimited script uploads (default cap is lower)
- Luraph obfuscation
- PandaAI assistant
- Live executions monitoring
- Import/export transfer features

## How To Upload To VSS

<SectionMarker icon={Upload} label="Upload Workflow" />

1. Select your service

   Click **Upload Script** and choose the service the script belongs to.

2. Upload your script file

   - Upload `.lua` or `.txt`
   - Or paste script code directly
   - Maximum file size: `40MB`

3. Configure script settings

   Available options include:

   - **Reinforced Loader**: block browser-based execution
   - **Disable Panda Macro**: skip `PANDA_AUTHENTICATE` / `PANDA_GETKEY` macro expansion
   - **Remove All Print Functions**: strip `print()` calls before obfuscation
   - **Whitelist Only**: allow only whitelisted Roblox players
   - **Discord Bot Visible**: allow bot delivery via Get Script features

4. Choose obfuscation

   - **IB1 (Free)**: fast, broad compatibility
   - **Luraph (Verified only)**: strongest protection, slower processing
   - **wYnfuscator (Beta, Free)**: advanced beta obfuscation
   - **None**: no obfuscation, fastest processing

5. Select library injection

   - **v4-rev-hyperguard**: non-GUI V4-HyperGuard, recommended for new projects
   - **V3-IVAL**: stable AES-256-CBC V3 library
   - **Legacy**: V2.5 compatibility and lightweight usage
   - **None**: obfuscation only, no library injection

## Getting Your Script

<SectionMarker icon={Download} label="Ready Output" />

After upload and processing, when script status is **READY**, you can:

- Copy the loadstring URL
- View analytics
- Check version history
- Edit settings
- Delete script

Use the link button to copy loadstring code:

```lua
loadstring(game:HttpGet("your-script-link"))()
```

## Version History And Rollback

<SectionMarker icon={History} label="Rollback Controls" />

VSS saves versions automatically.

Rollback flow:

1. Open version history
2. Select a version
3. Preview it
4. Click restore

Common use cases:

- Revert after introducing a bug
- Undo unwanted changes
- Compare versions
- Recover deleted code

Each version includes version number, timestamp, and auto-save notes (when available).

## Security Best Practices

<SectionMarker icon={Shield} label="Recommended Security Profile" />

Recommended for maximum security:

- Enable Reinforced Loader
- Use V4-HyperGuard
- Use Luraph or IB1
- Enable Whitelist Only for private scripts
- Remove print functions

For public/free scripts:

- Use IB1 or wYnfuscator
- Use V3 or V2.5 library
- Keep Discord bot visibility enabled
- Do not enable Whitelist Only for public access

## Tips And Tricks

<SectionMarker icon={Lightbulb} label="Operational Tips" />

Obfuscation speed expectations:

- `IB1`: around 30 seconds
- `wYnfuscator`: around 1-2 minutes
- `Luraph`: around 2-5 minutes

Token management tips:

- Upload/edit costs 1 token each
- Tokens reset monthly
- Batch edits when possible to reduce token burn

Library injection tips:

- Service ID is auto-configured
- Manual setup is usually not required
- Choose `None` only if you handle auth yourself
